The First Principle: Two-Way Web

A website isn't just a brochure; it must collect information from the user and send it to the server.

Without forms, you couldn't log in, search, buy products, or post comments. The web would be read-only.

The Core Problem

  1. Display interactive fields (text boxes, checkboxes, dropdowns).
  2. Package the user's data neatly.
  3. Send that package to a server.
  4. Tell the server how the data is sent.

The HTML solution: <form>

The <input> Element

Creates a field for user input. Behavior changes based on the type attribute. Basic type: text.

The <label> Element

Describes the input field so users know what to type.

Connecting label and input with id and for

Clicking the label focuses the input. Screen readers now correctly announce the label.
